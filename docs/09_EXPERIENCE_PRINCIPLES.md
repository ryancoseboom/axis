# 09_EXPERIENCE_PRINCIPLES.md

## Purpose

Axis is not designed to maximize engagement.

Axis exists to help someone spend today in a way that earns
self-respect.

These principles define how Axis should *feel*. When implementation
decisions are ambiguous, this document takes precedence over interface
novelty.

------------------------------------------------------------------------

# Core Principle

**Axis succeeds when it disappears.**

The best interaction is one that returns the user to meaningful work.

------------------------------------------------------------------------

# The Three States

## Morning --- Choose

Morning is reflective.

The user is invited to understand the shape of a good day, not to manage
an overwhelming list.

Morning should feel calm, spacious, and editorial.

It answers:

> "What deserves my life today?"

------------------------------------------------------------------------

## Begin --- Commit

Begin is a threshold.

It is neither motivational nor transactional.

It is the quiet moment where intention becomes action.

Crossing the threshold should reduce information, not increase it.

------------------------------------------------------------------------

## Now --- Do

Now protects attention.

Everything not needed for the current commitment disappears.

The interface becomes almost invisible.

It answers only one question:

> "What deserves my attention right now?"

------------------------------------------------------------------------

## Evening --- Learn

Evening is reflective, never judgmental.

Axis helps the user notice patterns, not assign grades.

The goal is wiser tomorrows, not perfect scores.

------------------------------------------------------------------------

# Reduction Over Addition

Every interaction should remove choices whenever possible.

Every transition should reduce cognitive load.

The interface becomes quieter as the day progresses.

------------------------------------------------------------------------

# Motion

Motion represents commitment, never excitement.

Transitions should feel architectural rather than animated.

The user should feel as though they have entered another room.

Avoid flashy effects, springs, celebrations, and reward animations.

------------------------------------------------------------------------

# Typography

Typography carries the emotional weight.

The interface should resemble a thoughtfully designed book more than a
dashboard.

Whitespace is functional.

Silence is part of the design.

------------------------------------------------------------------------

# Language

Axis never nags.

Axis never celebrates trivial accomplishments.

Axis speaks with clarity, restraint, and respect.

Prefer observations over instructions.

Prefer direction over motivation.

------------------------------------------------------------------------

# Information Hierarchy

Morning contains context.

Now contains only what protects the current commitment.

After Begin, information density should decrease dramatically.

------------------------------------------------------------------------

# Completion

Completion is not the goal.

Integrity is.

The purpose of Axis is not finishing tasks.

The purpose is spending today well.

------------------------------------------------------------------------

# Design Test

Every proposed feature should satisfy these questions:

1.  Does this reduce cognitive load?
2.  Does this increase self-respect rather than engagement?
3.  Does this help the user leave the app sooner?
4.  Would Dieter Rams remove it?
5.  Would it still feel appropriate ten years from now?
6.  Does it make an ordinary Thursday meaningfully better?

If the answer is "no," the feature should be reconsidered.

------------------------------------------------------------------------

# Final Principle

Every screen should quietly say:

> Close me when you know what matters.

When the user returns to life instead of interacting with Axis, the
product has succeeded.
